Question-2: What is the difference between innerHTML, innerText, and textContent ?

Answer:
innerHTML: Returns or sets the HTML content of an element, including both markup and text.
innerText: Returns or sets only the visible text content of an element, ignoring any HTML markup. It also considers styling.
textContent: Returns all the text content of an element, including text that is hidden or within script tags. It does not consider styling.

Question-3: What is event delegation in the DOM?

Answer: Event delegation is a technique of attaching a single event listener to a parent element to handle events that occur on its child elements. When an event is triggered on a child element, it bubbles up the DOM tree to the parent element, where the event listener can handle it. This reduces the need to add separate event listeners to many child elements, which can improve performance.

Question-4: What is event bubbling in the DOM?

Answer: Event bubbling is the process of event propagation in the DOM, where an event triggered on a child element bubbles up to its parent element, then to its parent's parent, and so on, until it reaches the root element (window or document). This is a fundamental concept for implementing event delegation.
